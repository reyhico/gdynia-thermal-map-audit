# Phase 2 Outlook

This document describes the planned scope of Phase 2 and identifies existing hooks
and placeholder code in the Phase 1 repository that support the transition.

---

## 1. Phase 2 Scope

Phase 2 extends the thermal-loss audit into a full explanatory / predictive study
by integrating socioeconomic, physical, and morphological covariates.

### 1.1 Socioeconomic Integration

| Variable | Source | Notes |
|---|---|---|
| Census income/education | GUS (Statistics Poland) | NUTS-5 or grid level |
| Building age | BDOT10k, OSM `start_date` | proxy via year of construction |
| Ownership type | Land registry / open data | public vs. private; cooperative |
| Population density | GUS, NSP 2021 | district / grid |
| Rent / property values | Scraped portal data | ethical review required |

### 1.2 Building Stock Characteristics

- Number of storeys (`building:levels`)
- Footprint area and volume proxy
- Wall material (brick, panel, prefab — `building:material`)
- Renovation history (where available)
- District heating connection (`network_adjacent_flag` from Phase 1 annotations)

### 1.3 Urban Morphology

- Floor Area Ratio (FAR)
- Sky View Factor (SVF) — computed from DSM if available
- Greenery fraction (NDVI from Sentinel-2)
- Street canyon geometry

### 1.4 Dependent Variable Definition

The Phase 2 dependent variable options:

1. **Continuous**: `mean_intensity` or `priority_index` at building / district level
2. **Binary**: `building_anomaly_flag` (anomaly vs. no anomaly)
3. **Ordinal**: `anomaly_scale_1_5` from annotations

The Phase 1 `SpatialUnitMetrics` and `BuildingLevelMetrics` schemas already include all
Phase 2 target variable fields.

### 1.5 Machine Learning Models

Planned models:
- Logistic regression (binary) — baseline
- Random forest / XGBoost (binary + ordinal)
- Spatial lag / error models (spatially explicit regression)
- Optional: deep learning on raster patches (if raster data available)

## 2. Existing Hooks in Phase 1

### 2.1 Schema hooks

`BuildingLevelMetrics` includes `has_anomaly`, `anomaly_scale`, and `area_m2` fields
that directly serve as Phase 2 response variables.

`SpatialUnitMetrics` includes `priority_index` that aggregates Phase 1 indicators and
will serve as a validation target for Phase 2 predictions.

### 2.2 Sampling hooks

`annotation.sampling.sample_buildings()` accepts `spatial_unit_gdf` for stratified
sampling by district — the same function will be used for Phase 2 survey design.

### 2.3 Spatial join hooks

`geodata.joins` provides generic `spatial_join` and `join_by_nearest` that will be
used to attach Phase 2 covariates to Phase 1 building-level records.

### 2.4 Grid infrastructure

The 100 m grid generated by `spatial_units.grid` provides a consistent spatial
support for joining heterogeneous data sources in Phase 2.

### 2.5 CLI extension points

The CLI in `cli.py` exposes placeholder hooks for:
- `compute-raster-indicators` — will be extended with multi-band raster processing
- `export-article-tables` — Phase 2 regression tables will be added here

## 3. 3D and Façade Module (Optional Exploratory)

Phase 2 optionally includes:

- **3D urban models**: CityGML LoD2 or 3D OSM building extrusions for volume estimation
- **Façade thermal analysis**: requires oblique imagery (not currently available)

These are marked as *optional* because:
- CityGML data for Gdynia may not be publicly available
- Façade IR imagery requires a separate acquisition campaign
- Analysis complexity is substantially higher

Placeholder module: `src/gdynia_thermal_audit/facade/` (not yet created; reserved namespace).

---

## 4. Ethical and Legal Considerations for Phase 2

Building-level socioeconomic data may be sensitive.  Phase 2 will require:
- Data protection assessment (GDPR compliance for any personal-level data)
- Ethical review from the researcher's institution
- Aggregation to minimum reporting unit before publication
