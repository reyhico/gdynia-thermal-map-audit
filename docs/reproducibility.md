# Reproducibility Guide

This document explains how to reproduce any pipeline run from scratch.

---

## 1. Environment Pinning

Exact package versions are frozen in `requirements-lock.txt` (generated by
`pip freeze > requirements-lock.txt` after a successful `make install-dev`).

To recreate the exact environment:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-lock.txt
```

Python interpreter version is recorded in every `PipelineRunLog` entry.

---

## 2. Configuration Snapshot

Every run hashes the active configuration file with SHA-256.  The hash is stored in
`PipelineRunLog.config_hash`.  The exact config file is copied to
`outputs/logs/config_snapshot_{run_id}.yaml` at run start.

---

## 3. Provenance Logging

### Source Inventory

Every downloaded file has a corresponding `SourceInventoryRecord` with:
- `checksum` (SHA-256 of the downloaded body)
- `timestamp` (UTC ISO 8601)
- `url` (canonical source URL)
- `lineage` (how the URL was discovered)

These records are appended to `data/interim/source_inventory.csv`.

### Fetch Log

Every HTTP request produces a `FetchLogRecord` appended to `data/interim/fetch_log.csv`.

### Run Log

Each pipeline invocation produces a `PipelineRunLog` JSON record saved to
`outputs/logs/run_{run_id}.json`.  Fields:

| Field | Description |
|---|---|
| `run_id` | UUID v4 |
| `start_time` | UTC ISO 8601 |
| `end_time` | UTC ISO 8601 |
| `scenario` | `raster`, `vector`, or `annotation` |
| `steps_completed` | List of completed pipeline step names |
| `errors` | List of error messages (if any) |
| `output_paths` | Dict of output file paths |
| `config_hash` | SHA-256 of config YAML |

## 4. Data Checksums

All files in `data/raw/` and `data/interim/` have SHA-256 checksums recorded in
`data/interim/checksums.csv`.  To verify integrity:

```bash
python -c "
from gdynia_thermal_audit.utils.hashing import sha256_file
import csv, pathlib
rows = list(csv.DictReader(open('data/interim/checksums.csv')))
for row in rows:
    p = pathlib.Path(row['path'])
    if p.exists():
        actual = sha256_file(p)
        status = 'OK' if actual == row['sha256'] else 'MISMATCH'
        print(f'{status}: {p}')
    else:
        print(f'MISSING: {p}')
"
```

## 5. Rerun Instructions

### Full pipeline rerun

```bash
cp .env.example .env      # configure environment
make install-dev
make run-pipeline
```

### Specific scenario

```bash
gta run-pipeline --scenario raster
gta run-pipeline --scenario vector
gta run-pipeline --scenario annotation
```

### Demo run (synthetic data, no network access)

```bash
gta run-pipeline --demo
# or
make demo
```

---

## 6. Notebook Reproducibility

Notebooks in `notebooks/` are deterministic when run top-to-bottom with a fresh kernel.
They read from `data/` and `data/demo/` and write nothing to `data/`.
Outputs are written to `outputs/`.

To execute all notebooks non-interactively:

```bash
jupyter nbconvert --to notebook --execute notebooks/*.ipynb --output-dir outputs/executed_notebooks/
```

---

## 7. Seed and Randomness

Any sampling operation (`annotation.sampling.sample_buildings`) accepts a `random_state`
parameter.  The default seed is recorded in the run log.  Set `PYTHONHASHSEED=0` for
full reproducibility of dict/set iteration order.

---

## 8. Citation

When citing derived results, include:
1. The run ID from `PipelineRunLog`.
2. The SHA-256 of the config file.
3. The package version (`gta --version`).
4. The date of data acquisition (from `source_inventory.csv`).
